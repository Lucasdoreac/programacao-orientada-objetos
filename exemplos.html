Exemplos Práticos

Veja exemplos práticos de POO em diferentes linguagens de programação. Cada exemplo demonstra como implementar e utilizar os conceitos de Programação Orientada a Objetos.

Sistema de Gestão de Biblioteca em Java

Este exemplo mostra um sistema básico de biblioteca com classes para livros, usuários e empréstimos, demonstrando encapsulamento, herança e polimorfismo.

// Classe Livro
public class Livro {
    private String titulo;
    private String autor;
    private String isbn;
    private boolean disponivel;
    
    public Livro(String titulo, String autor, String isbn) {
        this.titulo = titulo;
        this.autor = autor;
        this.isbn = isbn;
        this.disponivel = true;
    }
    
    // Getters e Setters
    public String getTitulo() { return titulo; }
    public String getAutor() { return autor; }
    public String getIsbn() { return isbn; }
    public boolean isDisponivel() { return disponivel; }
    public void setDisponivel(boolean disponivel) { this.disponivel = disponivel; }
    
    @Override
    public String toString() {
        return titulo + " por " + autor + " (ISBN: " + isbn + ")";
    }
}

// Hierarquia de usuários
public abstract class Usuario {
    protected String nome;
    protected String id;
    protected List livrosEmprestados;
    
    public Usuario(String nome, String id) {
        this.nome = nome;
        this.id = id;
        this.livrosEmprestados = new ArrayList<>();
    }
    
    public abstract int getLimiteEmprestimos();
    
    public boolean podeEmprestar() {
        return livrosEmprestados.size() < getLimiteEmprestimos();
    }
    
    // Getters
    public String getNome() { return nome; }
    public String getId() { return id; }
    public List getLivrosEmprestados() { return livrosEmprestados; }
}

// Subclasses de Usuário
public class Estudante extends Usuario {
    public Estudante(String nome, String id) {
        super(nome, id);
    }
    
    @Override
    public int getLimiteEmprestimos() {
        return 3; // Estudantes podem emprestar até 3 livros
    }
}

public class Professor extends Usuario {
    public Professor(String nome, String id) {
        super(nome, id);
    }
    
    @Override
    public int getLimiteEmprestimos() {
        return 10; // Professores podem emprestar até 10 livros
    }
}

Neste exemplo, vemos os seguintes conceitos de POO:

  • Encapsulamento: Atributos privados com getters e setters
  • Herança: Classes Estudante e Professor estendem a classe Usuario
  • Polimorfismo: Método abstrato getLimiteEmprestimos() implementado diferentemente nas subclasses
  • Abstração: Classe abstrata Usuario define a interface para diferentes tipos de usuários

Sistema de E-commerce em Python

Este exemplo demonstra um sistema de e-commerce básico em Python, mostrando como criar classes para produtos, carrinho de compras e clientes.

class Produto:
    def __init__(self, id, nome, preco, estoque=0):
        self.id = id
        self.nome = nome
        self.preco = preco
        self.estoque = estoque
    
    def esta_disponivel(self):
        return self.estoque > 0
    
    def reduzir_estoque(self, quantidade):
        if quantidade <= self.estoque:
            self.estoque -= quantidade
            return True
        return False
    
    def __str__(self):
        return f"{self.nome} - R${self.preco:.2f}"


class ItemCarrinho:
    def __init__(self, produto, quantidade):
        self.produto = produto
        self.quantidade = quantidade
    
    def subtotal(self):
        return self.produto.preco * self.quantidade
    
    def __str__(self):
        return f"{self.quantidade}x {self.produto.nome} - R${self.subtotal():.2f}"


class CarrinhoCompras:
    def __init__(self):
        self.itens = []
    
    def adicionar_item(self, produto, quantidade=1):
        # Verifica se o produto já existe no carrinho
        for item in self.itens:
            if item.produto.id == produto.id:
                item.quantidade += quantidade
                return True
        
        # Senão, adiciona novo item
        if produto.reduzir_estoque(quantidade):
            novo_item = ItemCarrinho(produto, quantidade)
            self.itens.append(novo_item)
            return True
        return False
    
    def calcular_total(self):
        return sum(item.subtotal() for item in self.itens)
    
    def __str__(self):
        if not self.itens:
            return "Carrinho vazio"
        
        resultado = "Itens no carrinho:\n"
        for item in self.itens:
            resultado += f"- {item}\n"
        resultado += f"\nTotal: R${self.calcular_total():.2f}"
        return resultado

Este exemplo em Python demonstra:

  • Encapsulamento: Métodos para controlar o acesso aos atributos (como reduzir_estoque)
  • Composição: CarrinhoCompras contém objetos ItemCarrinho
  • Sobrescrita de métodos: Método __str__ é sobrescrito para personalizar a representação em string

Sistema de Banco em C#

Este exemplo mostra um sistema bancário simples em C#, demonstrando encapsulamento, herança e polimorfismo com diferentes tipos de contas bancárias.

using System;
using System.Collections.Generic;

// Classe base para contas bancárias
public abstract class ContaBancaria
{
    // Atributos encapsulados
    public string Numero { get; private set; }
    public string Titular { get; set; }
    protected decimal Saldo { get; set; }
    
    // Construtor
    public ContaBancaria(string numero, string titular, decimal saldoInicial = 0)
    {
        Numero = numero;
        Titular = titular;
        Saldo = saldoInicial;
    }
    
    // Método para visualizar saldo (público)
    public decimal ConsultarSaldo()
    {
        return Saldo;
    }
    
    // Método para depósito
    public virtual void Depositar(decimal valor)
    {
        if (valor <= 0)
            throw new ArgumentException("O valor do depósito deve ser positivo");
            
        Saldo += valor;
        Console.WriteLine($"Depósito de {valor:C} realizado com sucesso.");
    }
    
    // Método abstrato para saque (cada tipo de conta implementa sua própria regra)
    public abstract bool Sacar(decimal valor);
    
    // Método para transferência
    public virtual bool Transferir(ContaBancaria contaDestino, decimal valor)
    {
        if (Sacar(valor))
        {
            contaDestino.Depositar(valor);
            return true;
        }
        return false;
    }
}

// Classe para conta corrente
public class ContaCorrente : ContaBancaria
{
    public decimal LimiteChequeEspecial { get; private set; }
    
    public ContaCorrente(string numero, string titular, decimal saldoInicial = 0, decimal limiteChequeEspecial = 0)
        : base(numero, titular, saldoInicial)
    {
        LimiteChequeEspecial = limiteChequeEspecial;
    }
    
    public override bool Sacar(decimal valor)
    {
        if (valor <= 0)
        {
            Console.WriteLine("O valor do saque deve ser positivo");
            return false;
        }
        
        // Verifica se o saque é possível com o limite do cheque especial
        if (valor <= Saldo + LimiteChequeEspecial)
        {
            Saldo -= valor;
            Console.WriteLine($"Saque de {valor:C} realizado com sucesso.");
            
            if (Saldo < 0)
                Console.WriteLine($"Atenção: Você está utilizando {Math.Abs(Saldo):C} do seu cheque especial.");
                
            return true;
        }
        else
        {
            Console.WriteLine("Saldo insuficiente para o saque");
            return false;
        }
    }
}

Este exemplo em C# demonstra:

  • Encapsulamento: Propriedades com modificadores de acesso (get; private set;)
  • Herança: ContaCorrente herda de ContaBancaria
  • Polimorfismo: Método virtual Depositar e método abstrato Sacar
  • Abstração: Classe abstrata ContaBancaria define estrutura básica
