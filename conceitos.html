Conceitos Fundamentais

A Programação Orientada a Objetos é um paradigma baseado no conceito de "objetos", que podem conter dados e código: dados na forma de campos (atributos) e código na forma de procedimentos (métodos).

O que é POO?

Programação Orientada a Objetos (POO) é um paradigma de programação baseado no conceito de "objetos", que podem conter dados na forma de campos (atributos) e código na forma de procedimentos (métodos).

A POO foi desenvolvida para facilitar o desenvolvimento e manutenção de programas de software, oferecendo benefícios como:

  • Modularidade: o código fonte de um objeto pode ser escrito e mantido independentemente
  • Reusabilidade: objetos podem ser reutilizados em diferentes programas
  • Escalabilidade: princípios de POO facilitam gerenciar programas maiores
  • Manutenibilidade: isolamento de partes do código facilita fazer mudanças

Classes e Objetos

Uma classe é um modelo ou blueprint a partir do qual objetos são criados. Classes definem as características (atributos) e comportamentos (métodos) que seus objetos terão.

Um objeto é uma instância de uma classe. Quando uma classe é definida, nenhum recurso é alocado, mas quando instanciamos um objeto, a memória é alocada para armazenar os valores das variáveis.


// Definição de uma classe em Java
public class Carro {
    // Atributos
    private String marca;
    private String modelo;
    private int ano;
    
    // Construtor
    public Carro(String marca, String modelo, int ano) {
        this.marca = marca;
        this.modelo = modelo;
        this.ano = ano;
    }
    
    // Método
    public void acelerar() {
        System.out.println("O carro está acelerando!");
    }
}

// Criando objetos (instâncias)
Carro meuCarro = new Carro("Toyota", "Corolla", 2023);
meuCarro.acelerar(); // Chamando um método do objeto
