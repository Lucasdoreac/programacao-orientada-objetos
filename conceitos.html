Conceitos Fundamentais

A Programação Orientada a Objetos é um paradigma baseado no conceito de "objetos", que podem conter dados e código: dados na forma de campos (atributos) e código na forma de procedimentos (métodos).

O que é POO?

Programação Orientada a Objetos (POO) é um paradigma de programação baseado no conceito de "objetos", que podem conter dados na forma de campos (atributos) e código na forma de procedimentos (métodos).

A POO foi desenvolvida para facilitar o desenvolvimento e manutenção de programas de software, oferecendo benefícios como:

  • Modularidade: o código fonte de um objeto pode ser escrito e mantido independentemente
  • Reusabilidade: objetos podem ser reutilizados em diferentes programas
  • Escalabilidade: princípios de POO facilitam gerenciar programas maiores
  • Manutenibilidade: isolamento de partes do código facilita fazer mudanças

Classes e Objetos

Uma classe é um modelo ou blueprint a partir do qual objetos são criados. Classes definem as características (atributos) e comportamentos (métodos) que seus objetos terão.

Um objeto é uma instância de uma classe. Quando uma classe é definida, nenhum recurso é alocado, mas quando instanciamos um objeto, a memória é alocada para armazenar os valores das variáveis.


// Definição de uma classe em Java
public class Carro {
    // Atributos
    private String marca;
    private String modelo;
    private int ano;
    
    // Construtor
    public Carro(String marca, String modelo, int ano) {
        this.marca = marca;
        this.modelo = modelo;
        this.ano = ano;
    }
    
    // Método
    public void acelerar() {
        System.out.println("O carro está acelerando!");
    }
}

// Criando objetos (instâncias)
Carro meuCarro = new Carro("Toyota", "Corolla", 2023);
meuCarro.acelerar(); // Chamando um método do objeto

Atributos e Métodos

Atributos (ou propriedades, campos, variáveis de instância) são as características que definem um objeto. Eles representam o estado do objeto e geralmente são declarados como variáveis dentro da classe.

Métodos são as ações ou comportamentos que os objetos podem realizar. Eles representam o que o objeto pode fazer ou como pode interagir com outros objetos.

Atributos Métodos
Representam o estado Representam o comportamento
Declarados como variáveis Declarados como funções
Ex: nome, idade, altura Ex: andar(), falar(), calcular()
Armazenam informações Realizam operações

Mensagem e Comunicação

Em POO, objetos comunicam-se entre si através de mensagens. Uma mensagem é uma solicitação para que um objeto execute um de seus métodos.

Uma mensagem geralmente contém:

  • O receptor (objeto que receberá a mensagem)
  • O nome do método a ser executado
  • Os parâmetros necessários (se houver)

Exemplo de comunicação entre objetos

Considere um sistema de e-commerce onde um objeto Cliente pode adicionar um objeto Produto a um objeto CarrinhoDeCompras. O cliente envia uma mensagem ao carrinho (chamando seu método adicionarItem) passando o produto como parâmetro. Esta troca de mensagens permite que os objetos trabalhem juntos mantendo seu encapsulamento.

Construtores e Destrutores

Um construtor é um método especial que é chamado quando um objeto é instanciado. Sua função é inicializar os atributos do objeto e realizar qualquer configuração necessária.

Um destrutor (ou finalizador) é um método chamado quando um objeto está prestes a ser destruído. Em linguagens com gerenciamento automático de memória (como Java e C#), os destrutores são menos comuns e geralmente substituídos por outros mecanismos.


// Exemplo de construtor em Python
class Pessoa:
    def __init__(self, nome, idade):  # Construtor
        self.nome = nome
        self.idade = idade
        print(f"Objeto {nome} criado")
        
    def __del__(self):  # Destrutor
        print(f"Objeto {self.nome} destruído")
        
# Criando objeto
p1 = Pessoa("João", 25)  # Saída: Objeto João criado

# Quando o objeto for removido pelo coletor de lixo
# Saída: Objeto João destruído

Pacotes e Namespaces

Pacotes (ou namespaces) são mecanismos para organizar classes e evitar conflitos de nomes. Eles permitem agrupar classes relacionadas e fornecer níveis adicionais de controle de acesso.


// Exemplo de pacotes em Java
package com.minhaempresa.utils;

public class StringFormatter {
    public static String capitalizar(String texto) {
        if (texto == null || texto.isEmpty()) {
            return texto;
        }
        return texto.substring(0, 1).toUpperCase() + texto.substring(1);
    }
}

// Em outro arquivo:
import com.minhaempresa.utils.StringFormatter;

public class App {
    public static void main(String[] args) {
        String nome = "joão";
        System.out.println(StringFormatter.capitalizar(nome));  // "João"
    }
}

Sobrecarga de Métodos e Operadores

A sobrecarga de métodos permite definir múltiplos métodos com o mesmo nome, mas diferentes parâmetros (tipo ou número). Isso aumenta a flexibilidade e legibilidade do código.

A sobrecarga de operadores (disponível em algumas linguagens) permite redefinir o comportamento de operadores como +, -, * para trabalhar com tipos definidos pelo usuário.


// Exemplo de sobrecarga de métodos em Java
public class Calculadora {
    // Sobrecarga do método somar
    public int somar(int a, int b) {
        return a + b;
    }
    
    public double somar(double a, double b) {
        return a + b;
    }
    
    public int somar(int a, int b, int c) {
        return a + b + c;
    }
}

// Uso
Calculadora calc = new Calculadora();
int resultado1 = calc.somar(5, 10);           // Chama a primeira versão
double resultado2 = calc.somar(2.5, 3.5);     // Chama a segunda versão
int resultado3 = calc.somar(1, 2, 3);         // Chama a terceira versão
