Os Quatro Pilares da POO

A Programação Orientada a Objetos se baseia em quatro princípios fundamentais que estabelecem as bases para criar software bem estruturado, reutilizável e de fácil manutenção.

1. Encapsulamento

O encapsulamento é o princípio de ocultar os detalhes internos de um objeto e expor apenas o necessário. É a prática de manter os atributos privados e fornecer métodos públicos para acessá-los (getters e setters).

Benefícios do encapsulamento:

  • Proteção de dados sensíveis
  • Controle sobre como os dados são acessados e modificados
  • Flexibilidade para alterar a implementação interna
  • Redução de dependências entre componentes

// Exemplo de encapsulamento em Java
public class ContaBancaria {
    private double saldo; // Atributo privado
    
    // Métodos públicos para acessar e modificar o saldo
    public double getSaldo() {
        return saldo;
    }
    
    public void depositar(double valor) {
        if (valor > 0) {
            saldo += valor;
        }
    }
    
    public boolean sacar(double valor) {
        if (valor > 0 && saldo >= valor) {
            saldo -= valor;
            return true;
        }
        return false;
    }
}

2. Herança

A herança permite que uma classe (subclasse) herde atributos e métodos de outra classe (superclasse). Este mecanismo promove a reutilização de código e estabelece uma relação "é-um" entre classes.

Tipos de herança:

  • Herança simples (uma classe herda de uma superclasse)
  • Herança multinível (classe C herda de B que herda de A)
  • Herança múltipla (uma classe herda de várias superclasses - não suportada em todas as linguagens)

// Exemplo de herança em Java

// Superclasse
public class Animal {
    protected String nome;
    
    public Animal(String nome) {
        this.nome = nome;
    }
    
    public void comer() {
        System.out.println(nome + " está comendo.");
    }
}

// Subclasse
public class Cachorro extends Animal {
    private String raca;
    
    public Cachorro(String nome, String raca) {
        super(nome); // Chama o construtor da superclasse
        this.raca = raca;
    }
    
    public void latir() {
        System.out.println(nome + " está latindo!");
    }
}

3. Polimorfismo

O polimorfismo permite que objetos de diferentes classes sejam tratados como objetos de uma classe comum. Literalmente significa "muitas formas" e permite que métodos tenham comportamentos diferentes em classes diferentes.

Tipos de polimorfismo:

  • Sobrecarga (overloading): múltiplos métodos com o mesmo nome mas parâmetros diferentes
  • Sobrescrita (overriding): redefine um método herdado da superclasse
  • Polimorfismo de subtipo: um objeto de uma subclasse pode ser tratado como objeto da superclasse

// Exemplo de polimorfismo em Java

// Sobrescrita (override)
public class Gato extends Animal {
    public Gato(String nome) {
        super(nome);
    }
    
    @Override
    public void comer() {
        System.out.println(nome + " está comendo peixe.");
    }
}

// Uso de polimorfismo
Animal animal1 = new Animal("Bob");
Animal animal2 = new Cachorro("Rex", "Labrador");
Animal animal3 = new Gato("Felix");

animal1.comer(); // "Bob está comendo."
animal2.comer(); // "Rex está comendo."
animal3.comer(); // "Felix está comendo peixe."

4. Abstração

A abstração é o processo de identificar apenas as características relevantes de um objeto, ignorando os detalhes irrelevantes. Isso permite criar modelos simplificados que capturam a essência das entidades do mundo real.

Benefícios da abstração:

  • Simplificação da complexidade
  • Foco nas características essenciais
  • Redução de duplicação de código
  • Melhoria da manutenibilidade

// Exemplo de abstração em Java

// Classe abstrata
public abstract class FormaGeometrica {
    // Método abstrato - sem implementação
    public abstract double calcularArea();
    
    // Método concreto
    public void desenhar() {
        System.out.println("Desenhando a forma...");
    }
}

// Implementações concretas
public class Circulo extends FormaGeometrica {
    private double raio;
    
    public Circulo(double raio) {
        this.raio = raio;
    }
    
    @Override
    public double calcularArea() {
        return Math.PI * raio * raio;
    }
}
